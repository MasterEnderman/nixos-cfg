# nixos-cfg

NixOS configuration using flake-parts, import-tree, disko, preservation, and noctalia-shell.

## Structure

```
flake.nix
modules/
    hosts/
        hp-probook/
        default.nix                  ← host declaration
        hardware.nix                 ← wraps hardware-configuration.nix + disko.nix
        hardware-configuration.nix   ← generated by nixos-generate-config
        disko.nix                    ← disk layout
features/
    common.nix                       ← shared system config
    niri.nix                         ← compositor + greetd
    preservation.nix                 ← tmpfs root + persistent state
    kernel.nix                       ← cachyos kernel
    git.nix                          ← system-level git
home/
    enderman/
        default.nix                  ← base user config
        git.nix
        noctalia.nix
        terminal.nix
```

## Installation

### 1. Boot the NixOS ISO

Download the minimal ISO from https://nixos.org/download and flash it to a USB drive:

```bash
dd if=nixos-minimal-*.iso of=/dev/sdX bs=4M status=progress
```

Boot the target machine from the USB. Switch to a root shell:

```bash
sudo -i
```

### 2. Connect to the internet

```bash
# wired — usually works automatically via DHCP
ip a

# wifi
iwctl
  station wlan0 scan
  station wlan0 connect "SSID"
  exit
```

### 3. Clone the config

```bash
nix-shell -p git
git clone https://github.com/MasterEnderman/nixos-cfg /mnt/etc/nixos
```

> The repo is cloned to `/mnt/etc/nixos` so it ends up at `/etc/nixos` after install, which is the bind-mount preservation creates for that path.

### 4. Partition and format with disko

> ⚠️ This **wipes `/dev/nvme0n1` completely**. Double-check the device path in `modules/hosts/hp-probook/disko.nix` matches your actual disk before running this.

```bash
nix --extra-experimental-features "nix-command flakes" run github:nix-community/disko/latest -- \
  --mode destroy,format,mount \
  /mnt/etc/nixos/modules/hosts/hp-probook/disko.nix
```

This creates:
- `/boot` — EFI partition (vfat, 512M)
- `cryptroot` — LUKS-encrypted btrfs partition
  - `@nix` → `/nix`
  - `@persist` → `/persist`
  - `@swap` → `/.swapvol` (8G swapfile)
- `/` — tmpfs (2G, wiped on every reboot)

### 5. Regenerate hardware configuration

```bash
nixos-generate-config --root /mnt --show-hardware-config \
  > /mnt/etc/nixos/modules/hosts/hp-probook/hardware-configuration.nix
```

Review the output before saving — make sure the initrd kernel modules look correct for your hardware.

### 6. Check before installing

Go through this list before running `nixos-install`:

- [ ] `disko.nix` — device path matches your actual disk (`/dev/nvme0n1` or adjust)
- [ ] `hardware-configuration.nix` — regenerated in step 5 and looks sane
- [ ] `features/common.nix` — timezone and locale are correct
- [ ] `features/niri.nix` — keyboard layout is correct (`"de"` by default)
- [ ] `home/enderman/git.nix` — name and email are correct
- [ ] `features/kernel.nix` — kernel variant is what you want (`cachyos-bore` by default)
- [ ] `hosts/hp-probook/default.nix` — LUKS device label matches the partition disko created (`disk-main-luks`)

### 7. Install

```bash
nixos-install --flake /mnt/etc/nixos#hp-probook --root /mnt
```

This will ask you to set a root password at the end. Set something temporary — you can disable root login after first boot.

### 8. Set the user password

```bash
nixos-enter --root /mnt -c 'passwd enderman'
```

### 9. Reboot

```bash
reboot
```

Remove the USB drive when the screen goes dark.

## First boot

### What to expect

1. GRUB hands off to systemd-boot
2. You are prompted for the **LUKS passphrase** to decrypt the drive
3. systemd brings up the tmpfs root and bind-mounts `/persist`, `/nix`
4. `greetd` + `tuigreet` presents a login prompt
5. Log in as `enderman` — niri starts, noctalia-shell launches on top

### First things to do

**Push your (possibly modified) config back to the repo:**

```bash
cd /etc/nixos
git add .
git commit -m "initial hp-probook configuration"
git push
```

> `/etc/nixos` is preserved across reboots via the preservation module, so your config survives the tmpfs wipe.

**Verify persistence is working:**

```bash
# should show your btrfs subvolumes bind-mounted under /persist
findmnt | grep persist
```

**Verify the kernel:**

```bash
uname -r
# should include "cachyos" or "bore" in the name
```

**Set up SSH keys** (if not restoring from backup):

```bash
ssh-keygen -t ed25519 -C "enderman@hp-probook"
# add ~/.ssh/id_ed25519.pub to GitHub
```

> SSH keys are listed in the preservation module and will survive reboots once created.

**Disable root login** (optional but recommended):

Add to `modules/features/common.nix`:

```nix
users.users.root.hashedPassword = "!";
```

Then rebuild:

```bash
sudo nixos-rebuild switch --flake /etc/nixos#hp-probook
```

## Rebuilding after changes

```bash
sudo nixos-rebuild switch --flake /etc/nixos#hp-probook
```

> Always `git add .` before rebuilding — import-tree only sees files tracked by git.
